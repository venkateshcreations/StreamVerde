🔲 Borders & Radius

Two border intensities and three radius sizes to create the layered glass-like aesthetic.

--radius (14px)

Cards, modals, panels

--radius-sm (8px)

Buttons, inputs, nav items

--radius-xs (6px)

Tiny action buttons, tags

Border tokens: --border (10% opacity) for subtle dividers, --border2 (22% opacity) for hover states and emphasis. Both use rgba(0,255,135,x).

📱 Sidebar & Topbar Specs

The app shell consists of a fixed sidebar and a sticky topbar.

Sidebar
Width: var(--sidebar-width) → 240px Collapsed: 60px icon rail (body.sidebar-collapsed) Background: rgba(0, 8, 4, .95) Border: 1px solid var(--border) — right edge Z-index: var(--z-sidebar) → 200 Transition: width .3s cubic-bezier(.25,.46,.45,.94) Nav item: .6rem .75rem padding, radius-sm, hover → surface2 Active: rgba(0,255,135,.1) bg + 2px left green border Tooltip: collapsed → ::after tooltip on hover (data-label attr) Badges: .badge (red pill) / .badge-g (green outlined pill)
Topbar
Height: var(--topbar-height) → 58px Background: rgba(0, 6, 3, .9) Blur: backdrop-filter: blur(20px) Position: sticky top:0, z-index var(--z-nav) Border: 1px solid var(--border) — bottom edge Components: hamburger | title | search bar | notif btn | avatar
Collapse behavior: On desktop, body.sidebar-collapsed shrinks sidebar to 60px icon rail with hover tooltips. On mobile (≤768px), sidebar is an overlay that slides from left. Labels always visible on mobile even in "collapsed" state.
💬 Modals & Overlays
Overlay: position fixed, inset 0, rgba(0,0,0,.75) backdrop-filter: blur(6px), z-index var(--z-modal) opacity 0 → .open: opacity 1 Modal box: background #001208, border var(--border2) max-width 480px, radius var(--radius) padding 1.75rem, max-height 90vh, overflow-y auto transform: translateY(20px) scale(.97) → open: none Components: .modal-header (flex between, title + close) .modal-form (flex column, gap .9rem) .modal-actions (flex end, gap .75rem, border-top)
Player modal uses max-width 640px, zero padding, black background with 16:9 aspect ratio player screen.
✨ Animations & Effects
/* Background orbs (ambient) */ @keyframes orbF { 0%,100% { translate(0,0) } 50% { translate(30px,-40px) } } Orb-1: 600px, rgba(0,255,135,.09) radial, top-left, blur 90px Orb-2: 500px, rgba(0,200,100,.06) radial, bottom-right, delay -10s /* Live indicator pulse */ @keyframes livePulse { 0%,100% { opacity:1 } 50% { opacity:.6 } } Duration: 2s ease-in-out infinite /* Player progress */ @keyframes playerProgress { 0% { width:0% } 100% { width:100% } } Duration: 30s linear infinite /* Common transitions */ Cards: all .25s ease (border, transform, shadow) Nav items: all .2s ease (background, color) Sidebar: .3s cubic-bezier(.25,.46,.45,.94) Modals: .3s ease (opacity, transform)
Hover effects: Media cards lift -4px + scale(1.02). Channel cards lift -3px + shadow. Stat cards lift -2px + top border glow. All links underline on hover.

📁 File Reference

Complete map of all CSS and layout files in the project.

FilePurpose
shared/css/tokens.cssDesign tokens (colors, fonts, spacing, borders, radius, z-index), global reset, global link color, scrollbar styles. Shared across website and app.
app/css/auth.cssLogin/signup card styles, auth tabs, role selector, social buttons, auth form layout.
app/css/shell.cssApp shell: sidebar (expanded/collapsed/mobile), topbar, main area layout, notification panel, hamburger, all responsive shell behavior.
app/css/components.cssReusable UI components: cards, stat cards, tables, buttons, tags, status labels, avatars, toggles, modals, upload zones, progress bars, bar charts, settings rows, layout grids.
app/css/pages.cssPage-specific styles: hero banner, media cards, channel cards, EPG grid, search, profile, DVR, player overlay, background orbs, animations.
website/css/layout.cssMarketing website layout styles.
Import order matters: tokens.css → auth.css → shell.css → components.css → pages.css. Tokens must always load first as all other files reference its custom properties.
